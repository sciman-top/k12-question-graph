# 106 · 教师可见元数据预算

日期：2026-06-06。

## 1. 判断

AI 推荐：v0.1 不再增加普通教师认知预算。新增能力优先并入现有任务流，不新增默认入口，不新增治理状态词，不把元数据膨胀成教师表单。

## 2. 全局预算

| 项目 | v0.1 预算 | 说明 |
|---|---|---|
| 首页默认入口 | `4` | 只保留导入试卷、找题组卷、导入成绩、查看分析 |
| 每个主页面首屏主操作 | `<= 3` | 超过 3 个主按钮就要重排或下沉 |
| 每个工作流首步必填项 | `<= 5`，目标 `<= 3` | 先靠默认值、模板、自动识别 |
| 教师可见状态词集合 | `<= 5` 类 | `待确认 / 可继续 / 已完成 / 需重试 / 需管理员处理` |
| 题卡默认可见元数据 | `<= 6` 项 | 超出走展开或详情 |
| 错误页下一步建议 | `<= 3` 条 | 只说影响和下一步，不倒日志 |
| 普通教师默认可见治理词 | `0` | 不暴露 candidate / active / migration / rollback 等 |

## 3. 教师默认可见字段

### 题卡默认字段

- 题型
- 难度任务词
- 来源摘要
- 知识点摘要
- 分值
- 题图/公式/表格是否完整

### 导入流程默认字段

- 文件名
- 页数或任务规模
- 当前状态
- 待确认项数量
- 失败原因摘要
- 下一步动作

### 成绩导入默认字段

- 模板名
- 映射状态
- 异常行数量
- 题号匹配状态
- 导入批次状态

## 4. 只能在管理员或高级模式出现的字段

- `candidate`
- `reviewed`
- `active`
- `productionEligible`
- `migration`
- `rollback snapshot`
- `manifest`
- `source hash`
- `provider profile`
- `token / cached_tokens / cost`
- `schema version / prompt version`
- `audit trail`
- `owner_scope`

## 5. 允许翻译后的教师任务语言

| 内部语义 | 教师可见替代词 |
|---|---|
| `candidate/pending_review` | 需确认 |
| `draft/test` | 示例流程 / 预览结果 |
| `retry_waiting` | 稍后重试 |
| `active version` | 当前正式版本 |
| `fallback used` | 已用保守方式处理 |

## 6. 预算超标时的默认动作

1. 先合并入口，不新增首页入口。
2. 先减少必填项，不新增教师字段。
3. 先改成默认值、模板、自动识别或批量处理。
4. 仍超标时，下沉到高级模式或管理员面板。
5. 仍不能解决时，后置到 `Later`，不挤进 v0.1。

## 7. 与 UI 合同的关系

本预算是 UI 合同和教师术语收口的上位边界。任何页面只要突破本预算，就必须在 UI 合同、路线图和功能准入卡里显式说明原因。
