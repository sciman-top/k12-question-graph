# 28 · 功能范围审查与优化

审查日期：2026-05-02。减法复核更新：2026-05-04。教师低学习成本复核更新：2026-05-05。

本文件回答“功能是否需要增加、删减、修改、优化”。结论：需要优化，但不是扩大功能面，而是收紧 P0/P1、补齐教师高频闭环的保障性能力，并把高级平台能力明确后置。

## 1. 裁决结论

AI 推荐：保持 v0.1 教师闭环不变，新增少量保障性能力，修改容易误导为“全自动/全平台”的表述，后置所有在线考试、学生端、复杂标准、复杂 AI 和平台化能力。

理由：本项目最高原则是教师工作流效率最大化。功能越多不等于教师越省时；P0/P1 阶段最有价值的是证明文件、任务、数据库、人工确认、来源证据和备份恢复能闭环。

2026-05-04 复核追加裁决：先完成 I008 教师简洁模式硬收口，再继续扩大 J0/K0 以后能力。普通教师默认入口只保留导入、组卷、成绩、分析四条任务流；C002R 修订、映射审核、知识资产健康、存储清理、active/candidate/migration/rollback 等治理面下沉到管理员或代理证据层。

2026-05-05 复核追加裁决：功能需求、路线图、计划和任务清单必须继续做减法。已经完成或被新任务吸收的旧任务不得以独立 ID 留在计划中；普通教师可见面不仅不能出现精确内部词，也不能出现 `draft/test`、`draft 动态资产`、`medium/medium_hard`、`0.x` 难度区间、生产资格、状态枚举等语义变体；后台权限不得再把 `/api/admin/*` 和 `/internal/ai/*` 裸接口阻断与完整角色审计混为一个待办项。

2026-05-05 review finding 收口裁决：减法要求不能只停留在文档和 backlog。`I009/I010/O004B/P001` 必须进入 `tools/run-roadmap-guard.ps1` 的硬检查；普通教师界面可见的服务状态、难度、来源、状态和治理词必须由 UI contract 直接阻断。

| 类别 | 裁决 | 验收 |
|---|---|---|
| 保留在普通教师侧 | 导入试卷、找题组卷、导入成绩、查看分析 | 四入口可完成高频任务，标签和说明使用教师自然语言 |
| 移出普通教师侧 | C002R 修订治理、映射审核、知识资产健康、存储/缓存清理、备份回滚摘要 | 默认隐藏在管理员或高级工作台，不被教师视图 CSS 打开 |
| 修改文案 | `draft_test`、`synthetic fixture`、`productionEligible=false`、`candidate`、`active` | 改为“示例流程”“示例数据”“正式启用前预览”“需确认”等可理解表述 |
| 后置扩展 | 真实 AI 写入、真实学生数据、发布权限、复杂治理看板 | 等教师简洁合同、隐私准入、`/api/admin/*` 与 `/internal/ai/*` 角色守卫和回滚证据齐备后再进入 |
| 删除或合并计划噪声 | 旧 `C003-C007`、`D004-D010`、`E005-E006`、`F004-F008` | 不再作为独立任务推进，能力并入 C002/D/L/M/N 主线或删除 |
| 拆分完成态 | `O004` 裸接口阻断与角色审计 | `O004` 记录已完成的 fail-closed guard，`O004B` 继续承担角色、审计、UI 权限边界 |

## 2. 保留功能

这些功能直接服务教师 Word/Excel 工作流，应保留在 v0.1。

| 功能 | 阶段 | 保留理由 | 验收重点 |
|---|---|---|---|
| 文件上传与文件仓库 | P0 | 所有导入、来源回看、备份恢复的起点 | 大文件不入库，hash、mime、size、path 可查 |
| ImportJob/AIJob/ReviewQueue | P0/P1 | 让长任务可追踪、可重试、可接管 | 状态、错误、attempt、idempotency、lease 可验证 |
| 文档解析/OCR Adapter | P1 | 隔离工具波动，支持 Word/PDF/图片入口 | 输出稳定内部 JSON 与 diagnostics |
| 人工确认队列 | P1 | 让教师只处理异常项 | 合并、拆分、跳过、重跑、题图关联 |
| SourceRegion 来源回看 | P1 | 让题目可审计、可修复、可追责 | 原文件、页码、bbox、截图、Adapter 诊断 |
| 初中物理知识本体 | P2 | 支撑检索、组卷、分析 | 稳定 L1-L3，本体与教材/课标/地区映射分层 |
| 题库检索与题目卡片 | P4 | 教师找题高频刚需 | 全文、知识点、题型、难度、来源筛选 |
| 自然语言组卷与细目表 | P4 | 降低组卷表达和配置负担 | 显示系统理解、细目表草稿、一键换题 |
| Word/PDF 导出 | P4 | 教师最终仍交付可打印文档 | Word/WPS 可打开，公式/题图/表格不丢 |
| Excel 成绩导入与基础分析 | P5 | 连接现有登分工作流 | 字段映射复用、异常集中提示、可讲评输出 |
| 备份恢复与缓存治理 | P0/P6 | 数据资产安全底线 | manifest/hash、独立脚本、恢复演练 |

## 4. 需要修改的功能表述

| 原倾向 | 修改为 | 原因 |
|---|---|---|
| AI 自动入库 | AI 预处理 + 异常确认 + 人工可接管 | 避免承诺全自动准确，降低教师信任风险 |
| 知识图谱 | 稳定知识点本体 + 映射层 | v0.1 不需要图数据库或复杂推理 |
| 完整题库平台 | 教师工作流闭环 | 防止走向 Moodle/Open edX 式完整 LMS |
| 智能组卷 | 系统理解 + 细目表草稿 + 可撤销换题 | 让教师理解和控制结果 |
| 学情诊断 | 可讲评、可补弱、可分层的基础分析 | 避免过早引入复杂学习科学指标 |
| 模型路由可配置 | 内置模型路由，普通教师不可见 | 减少普通教师认知负担 |
| 难度数值和内部状态直出 | 难度中等/略难、正式启用前预览、需确认 | 普通教师不应理解 `medium_hard`、`0.55-0.7` 或 `draft/test` |
| 后台合同完成 = 权限完成 | 裸接口阻断已完成，角色与审计仍待办 | 避免误判 live readiness |
