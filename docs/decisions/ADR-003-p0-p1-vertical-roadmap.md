# ADR-003 · P0/P1 最小纵切优先

## Status

Accepted

## Context

原始路线图覆盖题库、AI、组卷、导出、学情、运维等完整 v0.1 闭环，但如果横向铺开，容易先建大量模型和页面，迟迟不能验证真实教师工作流。

## Decision

编码只从 P0/P1 开始：

```text
P0: 打开应用 -> 登录占位 -> 上传文件 -> 创建 ImportJob -> 写数据库 -> 文件入仓 -> 备份 manifest
P1: 上传试卷 -> 文档解析/OCR 占位 -> 页面预览 -> 异常确认队列 -> 单题入库 -> 来源回看
```

P2-P6 在 P1 验收前不得实现真实高级功能，只允许保留 schema、配置和文档预留。

## Rationale

- P0 验证工程骨架、数据路径、任务路径、文件路径和备份路径。
- P1 验证教师最痛的入库流程，并建立 AI/人工协同边界。
- 后续知识本体、AI、组卷、学情和协同都依赖可靠题目来源与人工确认闭环。

## Consequences

- 任务清单必须有依赖、验收和验证命令。
- “模型已建好”或“页面已画好”不能作为阶段完成。
- P1 前不得提前接入复杂 RabbitMQ、完整 QTI/CASE/OneRoster、在线考试或学生端。
