# 14 · 备份、恢复、迁移与灾备

## 1. 备份范围

备份不只是题库题目。必须包括：

```text
数据库
文件仓库
教师账号与偏好
标签/字段配置
知识点图谱
教材/考点映射
组卷模板
导出模板
Excel 映射模板
AI prompt / 规则版本
模型路由配置
人工审核记录
AI 反馈样本
成绩与学情数据
系统配置
```

敏感信息如 API Key、数据库密码不明文导出。恢复后可要求重新配置或加密备份。

## 2. 三种备份包

| 类型 | 内容 | 用途 |
|---|---|---|
| 轻量题库包 | 结构化题目、题图、公式、表格、知识点、必要截图 | 教师/备课组共享 |
| 完整题库包 | 轻量包 + 原始文件 + AI 日志 + 审核记录 | 迁移/灾难恢复 |
| 冷归档包 | 完整包 + 压缩原始资料 + hash 校验 | 学期/学年归档 |

## 3. 自动备份策略

```text
每天：数据库 pg_dump + 关键配置
每天：文件仓库增量复制
每周：完整备份包
每月：冷归档
每次升级前：强制恢复点
每月/每学期：恢复演练
```

## 4. 备份目标

- 本机另一个目录。
- 另一个磁盘。
- 外置硬盘。
- NAS。
- 局域网另一台电脑共享目录。

推荐先备份到本机，再复制到网络共享；网络失败时保留本机备份并提醒。

## 5. 备份流程

```text
创建备份锁
→ pg_dump 数据库
→ 复制文件仓库
→ 导出配置/模板/教师偏好/prompt/规则
→ 生成 manifest.json
→ 生成 checksums.sha256
→ 验证备份
→ 复制到目标目录
→ 清理过期备份
→ 写入日志
```

## 6. WinPE 应急恢复

Windows 无法进入时，用 WinPE/启动 U 盘进入磁盘，复制 `D:\KQG_Data\` 与最近备份目录。系统应生成 `KQG_EmergencyCopy.cmd/ps1`，放在数据目录和恢复 U 盘中。

## 7. 程序打不开时

提供独立脚本：

```text
tools/backup.ps1
tools/restore.ps1
tools/verify-backup.ps1
tools/export-pack.ps1
```

这些脚本不依赖 Web UI。

## 8. 恢复演练

每月检查：数据库备份、文件仓库、配置、hash、恢复脚本。没有经过恢复演练的备份不算可靠备份。

## 9. RPO/RTO 目标

默认目标先按个人/校内小规模部署设置，后续可按学校要求收紧。

| 场景 | RPO | RTO | 说明 |
|---|---:|---:|---|
| 日常误删/误改 | <= 24 小时 | <= 4 小时 | 依赖每日备份和回收站 |
| 升级失败 | 0 或接近 0 | <= 2 小时 | 升级前必须创建恢复点 |
| 主程序不可用 | <= 24 小时 | <= 4 小时 | 独立脚本可运行 |
| Windows 无法启动 | 最近一次成功备份 | 半天内 | WinPE 拷贝数据目录与备份目录 |

P0 只要求生成 manifest/hash 并能校验；P6 再做完整恢复演练。

## 10. 备份失败阻断

以下情况不得报告备份成功：

- `pg_dump` 失败或退出码非 0。
- 文件仓库清单生成失败。
- manifest schema version 缺失。
- `checksums.sha256` 缺失或校验不通过。
- 备份目标目录不可写。
- 敏感配置被明文写入可导出的备份包。
