# 01 · 功能需求文档 PRD

## 1. 产品定位

校本题谱是面向中小学教师与备课组的 AI 原生校本题库与学情诊断平台。系统以可版本化、可替换、可追溯的学科知识体系和教学资产为核心，支持 Word/WPS/PDF/图片试卷入库，AI/规则辅助切题、识别公式图片表格、给出知识点与地区考点建议、校验答案解析、辅助组卷与高质量导出；通过 Excel 小题分导入实现班级讲评、知识点掌握和分层练习建议。

一句话目标：把教师已经在 Word、PDF、图片和 Excel 中完成的工作，低摩擦转成可检索、可复用、可导出、可分析、可恢复的校本资产。

减法目标：功能不是越多越好。v0.1 默认低学习成本、低培训依赖、低术语暴露；普通教师只面对导入、组卷、成绩、分析四个任务入口，高级治理、模型路由、备份恢复、权限审计和证据细节下沉到管理员或系统层。

## 1.1 文档边界

本 PRD 只定义稳定的产品目标、用户、成功指标和范围控制；当前仓库级、非现场与现场阻断、Go / No-Go 结论请看 `docs/112_CurrentClosureStatus_20260609.md`、`docs/103_ExecutionControlBoard.md`、`docs/109_ReleaseGoNoGoCard.md` 和 `tasks/completion-state-dashboard.csv`。这些状态入口已在 2026-06-13 按 2026-06-11 最新 repo-side 证据刷新。

## 2. 目标用户

| 用户 | 目标 | 关键痛点 |
|---|---|---|
| 普通教师 | 快速找题、组卷、导出、登分、分析 | 不愿学习复杂系统，不能比 Word/Excel 更慢 |
| 备课组长 | 共建校本题库、审核题目、统一知识点体系 | 题库混乱，质量不稳定，教师标准不同 |
| 学校管理员 | 权限、备份、存储、账号、数据安全 | 数据丢失、权限混乱、备份不可用 |
| 系统维护者 | 低成本运行、可恢复、可迭代 | AI 成本、文件膨胀、缓存和备份管理 |

## 3. v0.1 成功指标

| 指标 | 目标 |
|---|---|
| 组卷效率 | 普通教师 10 分钟内得到可打印 Word/PDF 初稿 |
| 导入效率 | 教师只确认异常项，系统不要求逐题审核全部 AI 结果 |
| 导出可靠性 | Word/WPS 可打开，公式、题图、表格不丢失 |
| 成绩导入 | 常见 Excel 模板首次映射后可复用 |
| 学情分析 | 输出能直接用于讲评、补弱和分层练习 |
| 易上手 | 普通教师默认四入口，界面不直出 `draft/test`、`candidate`、`active`、`medium_hard`、数值难度区间或后台审计术语 |
| 可恢复性 | 数据库、文件仓库、配置、模板、prompt 和教师偏好可同包校验恢复 |
| 成本可见 | AI 任务记录模型、prompt/schema 版本、token、成本、缓存命中和人工修改 |
| 动态资产演进 | 知识点、标签、题型、教材/课标映射、评分标准、组卷规则和分析指标可版本化、可迁移、可回滚；映射必须兼容一对一、一对多、多对一和多对多 |

## 3.1 指标测量协议

所有成功指标必须按统一记录协议测量，不能只写“感觉可用”：

| 指标 | 样本口径 | 起止口径 | 通过条件 | 证据 |
|---|---|---|---|---|
| 组卷效率 | 代表性教师需求或代理需求，使用当前允许的数据集 | 从提交组卷需求到产出可下载工件，不含打印 | 中位数 `<= 10` 分钟；若超时必须记录阻断项 | `M006` / `S009-S010` / `P006` |
| 导入效率 | 黄金样本、授权或脱敏材料 | 从上传开始到可进入人工确认或保存题目 | 不允许要求教师逐题全量复核；必须记录总题数、待确认题数、重裁题数 | `B008` / `S003-S006` / `REAL005` |
| 导出可靠性 | Word/WPS/PDF 典型试卷 | 从导出前审校到工件打开验证 | 工件可打开，题号、公式、题图、表格不丢失 | `E004` / `S010` |
| 成绩导入复用 | 同一模板的两次以上导入 | 从选择模板到批次入库 | 首次映射后第二次不得再逐字段重配 | `F002` / `S011` |
| 学情可用性 | 讲评报告、班级摘要、分层建议 | 从成绩导入完成到生成可讲评输出 | 先由确定性指标支撑，AI 文案只能是 draft | `F003` / `S011` |
| 易上手 | 代理或真实教师首轮使用 | 从进入系统到完成目标流 | 不依赖长手册；关键路径只靠页内提示即可继续 | `I001-I010` / `P004` |
| 可恢复性 | 备份恢复演练 | 从开始 restore 到校验恢复结果 | 恢复过程可验证，主程序不可用时仍可执行 | `G001-G004` / `NS801-NS806` |

所有指标至少记录：`scenario_id`、`dataset_type`、`operator_role`、`start_ts`、`end_ts`、`manual_touches`、`retry_count`、`fallback_used`、`result_status`。

## 4. 教师效率验收模型

v0.1 的每个用户故事必须同时给出功能验收和教师效率验收。功能“能用”但教师更慢，不算通过。

| 验收项 | P0/P1 口径 | v0.1 口径 |
|---|---|---|
| 首次路径 | 能打开应用、上传文件、查看任务状态和错误 | 普通教师无需阅读手册即可完成导入、组卷、导出、登分 |
| 人工确认 | 只确认异常项或 worker stub 标出的待处理项 | 高置信度自动通过，低置信度集中确认 |
| 默认值 | 数据目录、学科、年级、导入任务有可见默认值 | 教师偏好、模板、Excel 映射自动复用 |
| 失败继续 | 上传/任务/备份失败保留文件、错误和重试入口 | AI/OCR/导出失败可人工接管，不丢来源证据 |
| 可恢复 | P0 backup manifest 覆盖数据库占位、文件仓库、配置 | 完整备份包可校验恢复，主程序不可用时脚本可运行 |

## 4.1 Automation-First 功能实现原则

所有功能故事默认先落到规则、脚本、专用 API/UI、稳定 Adapter、schema、SQL、hash/cache、typed client、模板或可复跑 contract。AI 和 AI agent 只能作为限定辅助：语义候选、复杂映射、高风险复核、低置信度样本分流或外层并行编排；不得替代来源证据、业务规则、验收脚本、人工审核、备份恢复或正式写入守卫。

每个新增或继续推进的功能必须能回答：

- 哪些步骤由确定性规则、脚本或专用功能完成。
- 哪些步骤允许 AI/agent 参与，输出是否只进入 `candidate`、`pending_review`、`draft/test` 或人工确认队列。
- 如果不用 AI/agent，教师或管理员是否仍能通过专用功能继续完成任务。
- 对应 evidence 命令是什么，失败时是否 fail-closed。

教师可见面和高风险 AI 边界分别由 `docs/106_TeacherVisibleMetadataBudget.md` 与 `docs/107_AITrustAndReviewContract.md` 约束。

## 5. v0.1 核心用户故事

### 5.1 试卷导入

教师上传一份 Word/PDF/图片试卷，系统自动解析、切题、识别题图/公式/表格、提取答案解析、标注知识点。教师只处理系统标出的异常项，确认后题目入库。

验收标准：

- 支持 Word/PDF/图片上传。
- 题目能显示原始页码和区域截图。
- 能处理跨页题、共用题图、答案解析分离。
- 高置信度题目可自动入库，低置信度进入确认队列。
- 导入失败时保留原始文件、错误原因、可人工框选/录入的继续路径。
- P1 阶段不承诺全自动准确切题，只验收“可预览、可修正、可保存、可回看来源”。

### 5.2 找题与组卷

教师可通过自然语言或筛选条件组卷。系统生成组卷初稿，显示知识点覆盖、题型数量、总分、难度分布。教师可一键换题、同类题替换、导出 Word/PDF。

验收标准：

- 10 分钟内组出一份可打印试卷。
- 一键换题保留当前版本的知识点、题型、难度、分值约束；这些约束发生版本迁移时必须有影响报告。
- 导出学生版、教师版、答案版。
- 导出结果 WPS/Word 可打开，公式和图片不丢失。
- 系统展示“理解后的组卷要求”和细目表草稿，不展示底层 prompt。

### 5.3 成绩导入与分析

教师上传 Excel 成绩表，系统自动识别字段和小题分，匹配试卷题号，生成基础学情分析、知识点薄弱点、分层练习建议。

验收标准：

- 支持总分导入和小题分导入。
- Excel 字段映射可自动保存为教师模板。
- 输出个人报告、班级 Excel 分析、班级 Word/PDF 摘要。
- 异常行、缺失字段、题号不匹配必须集中预览，不静默丢数据。

### 5.4 备份与恢复

系统自动备份数据库、文件仓库、教师偏好、模板、AI 规则和配置。即使主程序打不开，仍可运行独立脚本进行备份/恢复。Windows 进不去时，可用 WinPE 拷贝指定数据目录。

验收标准：

- 支持本机目录、另一个磁盘、局域网共享目录备份。
- 支持恢复演练和备份校验。
- 支持轻量题库包、完整题库包、冷归档包。
- 备份 manifest 和 hash 校验失败时不得报告成功。

## 6. v0.1 首批实现边界

v0.1 是完整教师闭环，但编码顺序必须先打 P0/P1。P0/P1 只要求可运行骨架和导入闭环，不要求一次做完组卷、导出和学情。

| 阶段 | 必须可运行的最小闭环 | 阶段退出门禁 | 不做 |
|---|---|---|---|
| P0 | 登录占位、上传文件、创建 ImportJob、写 PostgreSQL、文件入仓、健康检查、最小备份 manifest | build/test/contract/hotspot 全部有命令；无代码子项目必须记录 gate_na | OCR、AI 切题、真实组卷 |
| P1 | 文档解析/OCR Adapter 占位、页面预览、SourceRegion、ReviewQueue、人工切题校正、单题入库、来源回看 | 至少一份黄金样本可从上传进入确认队列并保存题目 | 全自动准确切题、复杂知识图谱、正式导出引擎 |

功能范围裁决见 `docs/28_FunctionScopeReview.md`。如果新功能未被归入 Must/Should/Later/Never，不得直接进入 P0/P1。

## 8. 非功能需求

| 类别 | 要求 |
|---|---|
| 部署 | Windows-first；本机开发，终态校本局域网服务器部署 |
| 性能 | 普通题库检索 < 2 秒；组卷初稿 < 30 秒；导出常规试卷 < 60 秒 |
| 可恢复 | 每日备份；升级前强制备份；恢复演练；WinPE 应急拷贝路径 |
| 安全 | 角色权限、上传文件检查、操作审计、备份访问控制 |
| 隐私合规 | A000A 前锁定部署辖区、学生数据责任方、外部 AI 数据边界、fixture 脱敏和题库来源授权规则 |
| 易用性 | 普通教师默认极简模式；高级功能折叠；可见文案使用任务语言而非治理、测试、模型或权限术语 |
| 成本 | 模型路由内置；本地 OCR/规则优先；AI 调用日志与成本统计 |

## 9. 风险与约束

| 风险 | 控制方式 |
|---|---|
| 文档解析/OCR 不稳定 | Adapter 隔离、保留原图和 SourceRegion、允许人工框选继续 |
| AI 结构化输出错误 | JSON Schema、置信度、人工确认、Evals、prompt/schema 版本化 |
| 题库维护负担膨胀 | 字段准入卡、个人标签不默认进校级统计、标签合并/停用/迁移 |
| 知识点/标签/题型/规则变更破坏历史 | 版本化领域资产、替换映射、影响分析、自动迁移建议和人工审核 |
| Word/WPS 兼容性 | OMML + 图片兜底、导出黄金样本回归、WPS 打开验证 |
| 备份不可恢复 | manifest、sha256、恢复演练、独立脚本、WinPE 拷贝路径 |
| 标准实现过早扩大范围 | QTI/CASE/OneRoster/Caliper 只做模型映射预留，不做完整导入导出 |
| 学生数据误入外部 AI/fixture | P0/P1 使用合成或匿名化样本；真实学生数据外传必须先锁定辖区、授权、脱敏和人工确认边界 |
