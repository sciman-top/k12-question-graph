# 00 · 项目最高准则与硬约束

审查版本：2026-05-02。

## 1. 项目名称

**校本题谱**

副标题：**AI 原生校本题库与学情诊断平台**

代码名：`K12QuestionGraph`

## 2. 最高硬约束

> **教师工作流效率最大化。**

外部资料和成熟题库/测评平台实践核对后的结论：该最高原则应保留，并升级为可验收的工程裁决标准。本项目的首要根本性原则不是“功能最多”“AI 最强”或“标准最全”，而是让教师在真实工作中更快、更稳、更少负担地完成任务。

任何功能、界面、AI 流程、数据模型、技术选型、字段设计、测试策略，都必须回答：

```text
它是否显著减少教师工作量？
它是否降低教师认知负担？
它是否比 Word + Excel 更快、更可靠？
它是否减少重复劳动？
它是否会增加题库维护负担？
```

不能明确提升教师效率的功能，默认后置。

## 3. 教师效率度量

后续 PRD、路线图和任务验收必须至少绑定以下指标。没有指标的“效率提升”不作为开发依据。

| 指标 | v0.1 目标 | 说明 |
|---|---:|---|
| 组卷耗时 | <= 10 分钟 | 从自然语言需求到可打印 Word/PDF 初稿 |
| 导入人工确认量 | 只处理异常项 | 禁止把 AI 结果逐题甩给教师确认 |
| 高频入口数 | 普通教师默认 4 个 | 导入试卷、找题组卷、导入成绩、查看分析 |
| 重复配置次数 | 首次后自动记忆 | 学科、年级、教材、模板、Excel 映射、AI 模式 |
| 新字段准入 | 必须证明用途 | 检索、组卷、分析、导出或治理至少命中一项 |
| 失败可接管 | 必须可人工继续 | AI/OCR/解析失败时保留手工入库和来源证据 |
| 数据可恢复 | 每个核心闭环可备份 | 数据库、文件仓库、配置、prompt、模板和教师偏好同备同验 |

这些指标优先于技术偏好。若某个设计更先进但增加教师确认、配置或维护成本，默认不进入 v0.1。

## 4. 功能准入卡

任何新功能、字段、页面、AI 调用或外部工具引入前，必须先通过教师效率准入卡。不能通过的需求默认进入 backlog，不进入 P0/P1。

| 准入问题 | 必须给出的证据 |
|---|---|
| 减少哪一步教师工作？ | 对应用户故事、当前 Word/Excel 步骤和减少的点击/录入/等待 |
| 是否增加确认负担？ | 人工确认范围、默认值来源、异常项筛选规则 |
| 是否增加维护负担？ | 字段用途、迁移/合并/停用方式、批量回滚路径 |
| 失败后教师如何继续？ | 人工接管入口、来源证据、错误提示和恢复动作 |
| 是否影响成本或隐私？ | 本地工具优先、模型路由、缓存、审计、数据留存位置 |

P0/P1 的准入更严格：只允许进入工程骨架、上传入仓、任务状态、人工确认、来源回看和备份恢复所必需的能力。

## 5. 一级准则

### 5.1 默认即好用

用户首次进入系统，不应先配置大量参数。系统应基于教师账号记住并自动使用：学科、年级、教材版本、导出模板、成绩表映射、常用组卷要求等默认值。

### 5.2 人工只处理异常

AI 高置信度结果自动通过；中置信度结果进入抽检池；低置信度、高影响、高价值内容进入人工确认队列。

### 5.3 少字段、少点击、少跳转

必填字段严格控制。普通教师界面只暴露高频操作。高级设置折叠。

### 5.4 可撤销、可追溯、可恢复

批量标签、换题、导入、删除、清理、导出、备份恢复均应可撤销或可回滚。每道题能回看来源文件、页码、区域截图、AI 处理结果、人工修改记录。

### 5.5 AI 成本受控

能用规则和本地工具解决的，不调用大模型；能用小模型解决的，不用强模型；能批处理的，不走同步高价调用；相同输入应缓存结果。

### 5.6 数据安全高于功能扩张

题库、成绩、教师偏好、知识图谱、备份配置都是资产。软件坏了、Windows 进不去、数据库损坏时，仍必须有应急恢复路径。

### 5.7 架构可扩展，界面不复杂

底层预留全学科、全学段、QTI/CASE/OneRoster 映射，但 v0.1 只实现初中物理核心闭环。

### 5.8 动态领域资产可演进

知识点、教材章节、课程标准、地区考点、题型、标签、难度/能力维度、评分标准、组卷规则、AI prompt/schema/model routing、文档解析 pipeline、分析指标、导出模板、隐私策略和学校组织结构都不是静态常量。系统必须把它们作为可版本化、可追溯、可迁移、可回滚的领域资产处理。

规则和 AI 可以先自动生成映射、替换和迁移建议，高置信度且低影响的一对一变更可自动应用；一拆多、多合一、低置信度、高影响、影响历史学情口径或生产规则的变更必须进入人工审核。正式资产不得覆盖草稿历史，必须通过版本、状态、来源证据和迁移报告进入生产流程。

## 6. 明确不做

v0.1 不做：在线考试、在线监考、防作弊、全学科正式支持、自动主观题阅卷、复杂 IRT、完整 QTI/CASE/OneRoster 导入导出、学生端/家长端、公网 SaaS。

## 7. 功能冲突裁决规则

| 冲突 | 裁决 |
|---|---|
| 功能完整性 vs 教师效率 | 教师效率优先 |
| 技术先进性 vs 可维护性 | 可维护性优先 |
| AI 自动化 vs 成本/可靠性 | 成本可控与可靠性优先 |
| 字段丰富性 vs 录入负担 | 降低录入负担优先 |
| 长期扩展 vs MVP 落地 | MVP 闭环优先，架构预留 |
| 动态领域变化 vs 静态常量 | 版本化资产与迁移优先 |

## 8. AI 推荐结论

AI 推荐：保留“教师工作流效率最大化”作为最高原则，并把它落实到 PRD 验收、路线图顺序、任务准入、UX 默认值、AI 成本控制和备份恢复门禁中。理由：Moodle、TAO、Open edX 等成熟系统证明题库、题型、标准、测评能力可以无限扩张；本项目真正的差异化不是再做一个完整在线考试平台，而是让教师现有 Word/Excel 工作流更快迁移到结构化、可恢复、可分析的本地/LAN 系统。
