# 25 · 功能准入标准

## 1. 功能准入卡

每个新功能必须填写：

```text
功能名称：
目标用户：
使用频率：每天/每周/每月/低频
节省时间：
是否比 Word/Excel 更快：是/否
是否增加教师填写负担：
是否可由默认值/模板/AI/规则自动完成：
工程复杂度：低/中/高
AI 成本：低/中/高/无
维护成本：低/中/高
数据安全影响：
备份恢复影响：
是否 MVP 必须：
不做的后果：
验收标准：
阶段归属：P0/P1/P2/P3/P4/P5/P6/Later/Never in v0.1
功能裁决：保留/修改/后置/不进 v0.1
失败接管路径：
来源证据：
```

## 2. 通过条件

高频、高价值、能明显省时间、人工负担小、成本可控、可测试、可回滚。

## 3. 自动后置条件

- 一年只用几次。
- 需要大量人工填写。
- 会引入复杂新数据结构。
- 不影响核心闭环。
- AI 成本高且结果不可靠。
- 会显著增加普通教师界面复杂度。

## 4. 自动不进 v0.1 条件

- 面向学生端、家长端或在线考试/监考，而不是教师 Word/Excel 工作流。
- 需要公网 SaaS、复杂部署或学校级系统对接才能体现价值。
- 需要完整标准认证、复杂 IRT、跨学科知识体系或大规模协同才能完成。
- 会让 P0/P1 无法在本机闭环验证。

## 5. 最低证据要求

功能进入开发前，至少要有：

- 对应 PRD 用户故事或任务编号。
- 教师效率理由。
- 阶段归属。
- 验收命令或手工验收样例。
- 失败接管与回滚方式。
