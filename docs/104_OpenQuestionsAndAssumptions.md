## 2.2 当前结论

截至 2026-06-09，本表中的 `OQ01-OQ08` 均**不需要新增新的顶层 CSV 任务**。原因不是它们不重要，而是它们已经被现有任务链覆盖；本轮新增的 `tasks/live-pilot-closeout-plan.csv` 只是对既有 `REAL005 / P001 / P003 / P005 / P006` 的更细执行拆解，不改变主线层级：

- 发布与试点链：`P001 -> P006`
- 运行形态与 AI 路由：`NS1301 -> NS1308`
- 历史教师简洁收口：`I008 -> I010`
- 标准互操作边界：`R007` / `NS1203`

当前更需要做的是：

1. 保持 `docs/104` 作为治理入口，避免同一未决事项在多个文档里漂移。
2. 在对应任务关闭时，同步把本表状态改为 `closed` 并补证据链接。
3. 若只是把既有 `P001/P003/P005/P006/REAL005` 拆成更细执行项，应优先使用 `tasks/live-pilot-closeout-plan.csv` 这类子计划，而不是新增新的顶层主线任务。
4. 若未来出现**现有任务无法承接的新裁决动作**，才允许新增 CSV 任务。

## 3. 当前共识性假设

- v0.1 仍然是初中物理单学科教师闭环，不扩成 LMS。
- 普通教师默认只看导入、组卷、成绩、分析四入口。
- AI 默认只产出 `draft/candidate/pending_review`，不直接写 `active`。
- 首个试点先解决离线优先、稳定导入、稳定导出、稳定备份恢复，再谈更多模型和标准对接。
- 现场前默认先完成 AI 视觉代理、artifact probe、route smoke、配置诊断和恢复演练；人工只处理授权、签字和真实环境事实。

## 4. 更新规则

1. 新的项目级未决事项必须写清：默认假设、关闭条件、负责人角色、最迟关闭点。
2. 已关闭事项不删除，改为 `closed` 并补证据链接。
3. 任何会改变发布、权限、互操作、AI 默认路由或教师可见面的决定，都必须先更新本表再改路线图。
4. 新增 CSV 任务前，必须先证明该事项无法被现有 `backlog / non-site / productization` 任务承接；否则只更新本表映射，不增新任务。
