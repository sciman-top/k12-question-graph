# 11 · 教师工作流与 UX 设计

## 1. UX 最高原则

教师工作流效率最大化。界面应少操作、少学习、少配置、少打扰、多默认、多自动、多撤销、多批量、多模板。

## 2. 首页

普通教师默认只看到 4 个入口：

```text
导入试卷
找题组卷
导入成绩
查看分析
```

高级入口折叠：知识图谱、标签治理、模型路由、备份策略、字段管理。

## 3. 简洁模式与高级模式

| 模式 | 用户 | 功能 |
|---|---|---|
| 简洁模式 | 普通教师 | 一键导入、自动组卷、换题、导出、成绩导入、基础分析 |
| 高级模式 | 备课组长/管理员 | 细目表、知识点权重、标签治理、模型路由、备份策略 |

## 4. 试题导入工作流

```text
上传文件
→ 系统自动处理
→ 只显示异常项
→ 教师快速确认/修正
→ 入库
```

不要让教师逐题确认全部。

## 5. 自然语言组卷工作流

```text
教师输入自然语言需求
→ AI 优化为“系统理解”
→ 显示结构化要求和细目表草稿
→ 教师确认/修改
→ 系统选题
→ 教师一键换题
→ 导出
```

不直接展示底层系统 prompt，展示“优化后的组卷要求”。

## 6. 组卷界面布局

```text
左侧：筛选栏
中间：候选题卡片
右侧：当前试卷结构/题篮
底部：预览、导出、保存
```

题目卡片显示：题型、题干前几行、题图缩略图、主知识点、难度、质量状态、最近使用、推荐理由。

## 7. 一键换题

换题保持：同知识点、同题型、相近难度、相近分值、近期未用、不与当前卷重复。所有换题可撤销。

## 8. 成绩导入工作流

```text
上传 Excel
→ 自动识别字段
→ 预览前 10 行
→ 教师确认一次
→ 保存映射模板
→ 生成分析
```

## 9. 个人默认值

TeacherPreference 自动记住：默认学科、年级、教材、组卷用途、导出模板、字体字号、Excel 映射、AI 模式、常用筛选。

## 10. 新手引导

内置示例：第一次导入试卷、第一次组卷、第一次导入成绩、第一次生成班级分析。用示例学习，不要求教师读长说明。

## 11. P0/P1 页面与状态清单

本阶段只做代理场景和工程验收，不要求真实教师现场验收。

| 页面 | P0/P1 必备状态 |
|---|---|
| 首页 | empty、loading、error、ready；普通教师只显示 4 个入口 |
| 上传页 | 文件选择、上传中、上传成功、类型不支持、文件过大、重复文件 |
| ImportJob 详情 | queued、running、succeeded、failed、retry_waiting、cancelled |
| 任务错误页 | 错误码、错误摘要、可重试/不可重试、保留原文件提示 |
| ReviewQueue | 无待处理、加载中、待确认列表、保存成功、保存失败 |
| 来源预览 | 页面加载中、页码跳转、区域高亮、截图缺失、原文件缺失 |
| 备份状态 | manifest 生成中、校验成功、校验失败、目标目录不可写 |

P0/P1 前端不做复杂视觉设计，优先保证状态明确、错误可理解、下一步动作可见。
