# 17 · 安全、隐私与合规

## 1. 数据类型

| 数据 | 风险 |
|---|---|
| 学生姓名、班级、学号、成绩、学情报告 | 个人信息/敏感教育数据 |
| 教师账号、偏好、操作日志 | 账号与行为数据 |
| 题库原始资料 | 可能涉及版权和来源限制 |
| AI prompt、反馈样本 | 可能含题目和学生数据 |
| 备份包 | 集中高风险资产 |

## 2. 权限角色

| 角色 | 权限 |
|---|---|
| 普通教师 | 个人题库、任教班级成绩、普通组卷 |
| 备课组长 | 学科组题库、审核、共享、模板 |
| 管理员 | 账号、备份、存储、安全、系统设置 |
| 审核员 | 校本题库审核 |

## 3. 安全基线

- 登录认证。
- 角色权限。
- API 权限校验。
- 上传文件类型和大小限制。
- 操作审计。
- 管理员高风险操作二次确认。
- 备份包访问控制。
- API Key 加密保存或恢复后重新配置。
- 导出报告可加水印。
- 学生报告可脱敏。

## 4. 版权与来源治理

每道题/资料记录来源：教师原创、校本试卷、真题、教材、网络、商业资料、未知。校本共享与导出传播应提示来源限制。

## 5. AI 数据策略

- 不把学生隐私无必要发送给外部模型。
- 学情报告生成只发送统计指标和必要上下文。
- AI 调用日志不保存不必要的敏感明文。
- 外部模型调用与成本可审计。

## 6. 合规辖区与学生数据边界

本文件不是法律意见。A000A 前必须记录实际部署辖区、学校数据责任方、数据处理者/受托方边界和外部模型传输边界；未锁定前按最严格保守策略执行。

| 场景 | P0/P1 默认策略 |
|---|---|
| 中国大陆学校/校内局域网 | 参考个人信息保护法等个人信息处理、最小必要、告知同意、敏感个人信息和未成年人数据保护要求；外部模型默认不接收学生身份和成绩明文 |
| 美国 K-12 或服务美国学校 | 参考 FERPA 教育记录、COPPA 13 岁以下在线服务、学校官方授权和家长/监护人同意要求 |
| 辖区未确定或跨境部署 | 禁止把真实学生身份、成绩、原始成绩表、含学生信息的 prompt 发送给外部模型；fixture 使用合成数据 |
| 教研/演示/回归样本 | 不使用真实学生姓名、学号、班级、成绩；真实试卷来源必须记录版权/授权和传播限制 |

A000A 必须输出最小数据边界：

```text
deployment_jurisdiction
data_controller_or_owner
operator_or_processor
student_pii_allowed_in_system: yes/no/limited
student_pii_allowed_in_external_ai: no by default
fixture_data_policy: synthetic_or_anonymized_only
backup_encryption_or_acl_policy
retention_and_delete_policy
```

## 7. 参考安全标准

v0.1 不做完整认证，但参考 OWASP ASVS 和 NIST AI RMF 思路：身份认证、授权、上传安全、日志、AI 风险识别、人工确认、可回滚。

## 8. P0/P1 威胁模型

| 风险 | 影响 | P0/P1 控制 |
|---|---|---|
| 上传恶意文件 | 解析工具执行风险、磁盘污染 | 文件类型 allowlist、大小限制、随机存储名、禁止把上传路径当命令参数拼接 |
| 越权读取原始试卷/成绩表 | 泄露题库或学生数据 | API 按角色/所属范围校验，文件下载不暴露真实磁盘路径 |
| 备份包泄露 | 集中泄露数据库、文件、prompt、成绩 | 备份目录权限、manifest 不含明文密钥、敏感配置恢复后重新录入 |
| AI prompt/log 泄露学生信息 | 外部模型或日志暴露隐私 | P0/P1 不接真实模型；后续 AI 调用默认脱敏和最小上下文 |
| API key/DB password 泄露 | 外部服务和数据库被滥用 | 不写入导出包；本机配置加密或恢复后重新配置 |
| 局域网弱账号 | 非授权访问系统 | P0 至少有登录占位和管理员/教师角色边界，P6 再强化策略 |
| 日志过度记录 | 错误日志含敏感正文 | 日志默认截断，避免保存完整学生成绩表、API key 和模型输入明文 |
| fixture 或 prompt 含真实学生数据 | 测试、日志、模型调用扩散隐私 | P0/P1 fixture 使用合成或充分匿名化数据；外部 AI 默认禁用真实学生数据 |
| 题库来源版权不清 | 校本共享或导出传播风险 | 每个 SourceDocument 记录来源、授权状态和传播限制；未知来源默认不可校级共享 |

## 9. P0 安全门禁

P0 gate 至少检查：

```text
合规辖区与学生数据边界已记录
上传类型和大小限制
API 未登录访问被拒绝
普通教师不能访问 admin 接口
文件下载不返回绝对磁盘路径
日志/备份 manifest 不含 API key 或数据库密码
fixture/prompt 不含真实学生姓名 学号 成绩 明文
外部 AI 调用在 P0/P1 默认禁用或只接收合成/脱敏数据
备份校验失败不报告成功
```
