# 02 · MVP 范围与范围控制

## 1. v0.1 冻结范围

v0.1 只做以下教师闭环：

```text
初中物理 → 文档入库 → 题库检索 → 自然语言组卷 → 导出 → Excel 成绩导入 → 基础学情分析 → 备份恢复
```

实现顺序必须收敛：先 P0 工程骨架，再 P1 多模态试题入库，不因路线图里存在 P2-P6 就提前实现高级能力。

功能范围的详细裁决见 `docs/28_FunctionScopeReview.md`。本文件只保留 v0.1 边界和准入规则。

## 2. v0.1 必做功能

| 编号 | 功能 | 说明 |
|---|---|---|
| M00 | 最小工程纵切 | 上传文件、创建任务、写库、文件入仓、健康检查、备份 manifest |
| M01 | 教师登录与偏好 | 默认学科、年级、教材、模板、Excel 映射 |
| M02 | 文件上传 | Word/PDF/图片，保留原始文件和优化文件 |
| M03 | 文档解析 | Docling/PaddleOCR/OpenXML 适配层 |
| M04 | AI/规则切题 | 题号、小问、答案解析、题图、表格、公式 |
| M05 | 人工异常确认 | 只处理低置信度项，不逐题确认全部 |
| M06 | 初中物理知识点本体 | 知识点稳定，课标/教材/考点为映射层 |
| M07 | 题库检索 | 全文检索 + 知识点/题型/难度/来源筛选 |
| M08 | 自然语言组卷 | 显示系统理解，生成细目表和试卷初稿 |
| M09 | 一键换题 | 同知识点、同题型、相近难度、近期未用 |
| M10 | Word/PDF 导出 | 学生版、教师版、答案版 |
| M11 | Excel 成绩导入 | 字段映射、总分/小题分、模板保存 |
| M12 | 基础学情分析 | 得分率、知识点掌握、题目难度、班级报告 |
| M13 | 备份恢复 | pg_dump + 文件仓库 + 配置 + 教师偏好 + 模板 |
| M14 | 缓存清理 | 临时文件、过期导出、失败任务现场 |
| M15 | 模型路由 | 内置 AI 任务路由、成本日志、缓存 |

## 3. v0.1 功能裁决摘要

| 类别 | 功能 |
|---|---|
| 保留 | 文件上传、文件仓库、ImportJob、ReviewQueue、SourceRegion、人工确认、题库检索、自然语言组卷、Word/PDF 导出、Excel 成绩导入、备份恢复 |
| 修改 | AI 自动入库改为 AI 预处理 + 异常确认；知识图谱改为稳定本体 + 映射层；模型路由普通教师不可见 |
| 增加 | 功能准入卡、导入黄金样本、人工接管路径、来源证据链、教师效率场景验收 |
| 后置 | 全学科、自动主观题阅卷、学生/家长端、完整标准互操作、复杂 IRT、微服务/RabbitMQ/Kubernetes/图数据库 |

## 4. P0/P1 先行边界

| 阶段 | 本阶段完成才算完成 | 明确不做 |
|---|---|---|
| P0 | 运行骨架、数据库迁移、文件仓库、ImportJob、AIJob/ReviewQueue 占位、健康检查、基础备份脚本、build/test gate | OCR 准确率、AI 自动切题、真实 Word/PDF 导出 |
| P1 | 文档解析/OCR Adapter、页面预览、SourceRegion、人工合并/拆分/题图关联、人工接管路径、QuestionItem 保存、来源回看、黄金样本占位 | 复杂知识图谱、自然语言组卷、成绩分析 |

P0/P1 的验收必须能在本机复跑，不允许只写界面或只写后端模型。

## 5. 明确后置

| 功能 | 后置原因 |
|---|---|
| 全学科正式支持 | 会严重扩张知识体系、题型、模板维护量 |
| 自动主观题阅卷 | 风险高，教师责任问题大 |
| 在线考试 | 不符合教师日常核心工作流 |
| 学生端/家长端 | 先把教师端闭环做好 |
| 完整 QTI/CASE/OneRoster 实现 | 数据模型预留即可 |
| IRT/等值 | 样本量和解释成本较高，先 CTT |
| 图数据库 | PostgreSQL 足够支撑 v0.1 |
| RabbitMQ/分布式队列 | P0/P1 任务量可由数据库持久化 job 和 BackgroundService 支撑 |
| 完整在线阅卷与监考 | 不符合教师 Word/Excel 核心工作流 |

## 6. 功能准入门禁

每个新功能进入开发前必须回答：

1. 教师多久用一次？
2. 能节省多少时间？
3. 是否明显比 Word/Excel 更快？
4. 是否增加题库维护负担？
5. 是否需要教师额外填写字段？
6. 是否能用默认值、模板、AI 或规则自动完成？
7. 没有它，核心闭环是否仍能运行？
8. 是否增加 AI token 成本？
9. 是否影响备份、恢复、权限、审计？

通过标准：高频、高价值、低额外负担、可测试、可回滚。

## 7. 防膨胀原则

- 普通教师界面默认只保留 4 个入口：导入试卷、找题组卷、导入成绩、查看分析。
- 高级能力隐藏：知识图谱、标签治理、AI 设置、模型路由、备份策略。
- 每个新字段都要证明它会被检索、组卷、分析或导出使用，否则不加入。

## 8. 标准预留边界

| 标准 | v0.1 允许 | v0.1 禁止 |
|---|---|---|
| QTI | QuestionItem/Paper/Result 映射字段预留 | 完整 QTI 导入导出和认证 |
| CASE | KnowledgeNode 与课程标准 GUID 映射预留 | 把课标当成本体主干 |
| OneRoster | Student/ClassGroup/ScoreRecord 字段预留 | 对接 SIS 或正式同步 |
| Caliper | AnalysisEvent/ActivityEvent 概念预留 | 建实时学习事件流 |
